**Learning objectives**

- Find the official repository (if any) for curriculum code and solutions.
- Know how to run and extend the exercises locally.

## Where to get the code

The curriculum is hosted at **GitHub** (see the edit link on each page: "Suggest Changes" points to the repo). Code snippets appear **inside the chapter pages** (exercises, hints, and worked solutions). You can:

1. **Copy from the site:** Type or copy the code from the exercise and solution sections into your own scripts or notebooks. This is the intended way to learn—you implement and run locally.
2. **Clone the repo (if a separate code repo exists):** If the project provides a dedicated code repository (e.g. `Reinforcement` or a `code` subfolder), clone it and run the examples. Check the [home page](/) or the repository README for the exact URL and setup instructions.
3. **Use your own code:** The exercises describe what to implement; you can write your own from scratch. The worked solutions are there to check your approach.

## Setup

To run the code you write (or clone), you need Python and the libraries used in the curriculum. See [Setting Up Your Environment](appendix/setting-up-environment/) and [Installing Libraries](appendix/installing-libraries/) in the Appendix. Use a virtual or conda environment so dependencies do not conflict with other projects.

## Contributing

If you have improvements or fixes, use the "Suggest Changes" link on a page to open the file on GitHub and submit a pull request. See the repository’s CONTRIBUTING or README for guidelines.
